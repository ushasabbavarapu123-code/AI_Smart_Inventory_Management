4. KEY DESIGN DECISIONS
--------------------------

Decision 1: Explicit Table Count Selection (8 Tables)
  - Rationale: The original DAY_03 requirements referenced "7 tables" in some lists, but the approved Database Design document (`DATABASE_DESIGN.md`) specifies 8 tables, adding the users and audit logs tables. We implemented all 8 tables to support future user auth and audit trail features.

Decision 2: SQLite Write-Ahead Logging (WAL) Mode
  - Rationale: Enabling WAL mode allows multiple concurrent readers (e.g. backend api routes and analytics notebooks) while writing operations occur, avoiding DB lock exceptions in development.
