4. KEY DESIGN DECISIONS
--------------------------

Decision 1: SQLite over PostgreSQL/MySQL
  Rationale: Local deployment scope, no installation overhead, fully
  sufficient for the project scale (50 products, 500+ sales records).

Decision 2: Node.js child process for Python forecasting
  Rationale: Decouples the ML layer from the web server. Python can
  run independently and write results back to the shared SQLite database.
  This avoids complex IPC and keeps each layer independent.

Decision 3: ARIMA as primary forecasting model
  Rationale: ARIMA is the industry standard for univariate time series
  demand forecasting. It handles trends and seasonality. A linear
  regression fallback is included for products with sparse data.

Decision 4: Vanilla JS frontend (no framework)
  Rationale: Internship project scope does not require React/Vue.
  Vanilla JS is lighter, loads faster, and demonstrates core JS skills.
